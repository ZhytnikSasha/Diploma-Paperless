# Веб-система для автоматизації документообігу «Paperless»

Цей репозиторій містить вихідний код кваліфікаційної (дипломної) роботи бакалавра на тему «Веб-система для автоматизації документообігу в невеликій організації». 

Система розроблена для спрощення внутрішнього документообігу: створення документів, управління їхніми статусами, маршрутизації між співробітниками та менеджерами, а також ведення незмінного журналу аудиту (історії дій).

## 🚀 Функціональні можливості
- **Рольова модель доступу:** Підтримка ролей «Адміністратор», «Менеджер» та «Співробітник».
- **Життєвий цикл документа:** Переходи статусів (`Чернетка` ➔ `На розгляді` ➔ `Затверджено` / `Відхилено`).
- **Робота з файлами:** Завантаження, редагування та безпечне зберігання фізичних файлів (PDF, DOCX тощо) на сервері.
- **Журнал аудиту:** Автоматичне фіксування всіх змін статусів та дій користувачів (сутність `DocumentHistory`).
- **Особистий кабінет:** Керування профілем, зміна пароля та встановлення аватара користувача.
- **Панель адміністратора:** Керування ролями співробітників та перегляд глобального журналу подій.

## 🛠 Технологічний стек
- **Мова програмування:** C#
- **Фреймворк:** ASP.NET Core MVC (.NET)
- **База даних:** MySQL
- **ORM:** Entity Framework Core (Code-First)
- **Автентифікація та Безпека:** ASP.NET Core Identity
- **Frontend-технології:** HTML5, CSS3, Bootstrap 5, JavaScript
- **Тестування:** xUnit, Moq, EF Core In-Memory Database

## ⚙️ Інструкція з розгортання та запуску

### 1. Вимоги до середовища
- Встановлений [.NET SDK](https://dotnet.microsoft.com/download) (відповідної версії).
- Встановлений та запущений сервер бази даних **MySQL**.
- Visual Studio 2022 або Visual Studio Code.

### 2. Клонування репозиторію
```bash
git clone [https://github.com/ZhytnikSasha/Diploma-Paperless.git](https://github.com/ZhytnikSasha/Diploma-Paperless.git)
cd Paperless
```

### 3. Налаштування бази даних
Відкрийте файл `appsettings.json` та вкажіть свої дані для підключення до MySQL у рядку `DefaultConnection`:
```json
"ConnectionStrings": {
  "DefaultConnection": "Server=localhost;Database=PaperlessDb;User=root;Password=ВашПароль;"
}
```

### 4. Застосування міграцій
Відкрийте консоль диспетчера пакетів (Package Manager Console) у Visual Studio або термінал і виконайте команду для створення бази даних та таблиць:
```bash
# Для .NET CLI:
dotnet ef database update

# Для Package Manager Console у Visual Studio:
Update-Database
```

### 5. Перший запуск та дані для входу
Система автоматично наповнить базу даних базовими ролями та створить обліковий запис головного адміністратора (логіка реалізована в `DbSeeder.cs`). 

Дані для входу адміністратора за замовчуванням:
- **Email:** `admin@paperless.local`
- **Пароль:** `AdminPassword123!`

## 📁 Структура проєкту
- `Controllers/` — Контролери MVC (обробка запитів та бізнес-логіка).
- `Models/` — Доменні моделі бази даних (`Document`, `ApplicationUser`, `DocumentHistory`).
- `Models/ViewModels/` — DTO-об'єкти для передачі даних у подання.
- `Data/` — Контекст бази даних Entity Framework (`AppDbContext.cs`).
- `Views/` — Шаблони сторінок інтерфейсу (Razor).
- `wwwroot/` — Статичні ресурси, завантажені файли документів (`/uploads`) та аватари (`/avatars`).
- `Paperless.Tests/` — Проєкт із модульними, інтеграційними, системними тестами та валідаційними.
